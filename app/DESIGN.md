# Test Ease — "Control Room" design system (#1822)

Dark instrument-panel aesthetic for the persona-QA workbench. One rule
above all: **severity is the only loud colour**. Cyan = interactive,
emerald = alive, red/amber = broken, everything else is quiet ink.

## Theming

Two themes ship as CSS-variable sets (`--te-*`) on `:root[data-theme]`,
generated by the plugin in tailwind.config.js. **Dark ("control room")
is the default**; light swaps the variable values only — no `dark:`
variants anywhere. The toggle lives in the sidebar (persisted under
`localStorage["testease.theme"]`, applied pre-paint by the index.html
boot script). Accent text shades 200/300/400 are var-driven too: dark
resolves them to Tailwind's pastels, light to the deep 800/700/600
equivalents — so `text-red-300` is readable on both. Shades ≥500
(washes `/10`, rings `/30`) are literal and work on both themes.
Hairlines use the `hairline` colour with an explicit alpha
(`border-hairline/[0.07]`) — white-alpha on dark, slate-alpha on light.
Never hardcode a hex in scoped CSS; use `rgb(var(--te-…))`.

## Tokens (tailwind.config.js)

- The `ink` scale is **flipped dark-side-up** in the default theme:
  `ink-50` is the deepest surface, `ink-900/950` the brightest text. Semantics of existing
  utilities are preserved — `text-ink-900` still means "highest
  contrast", `bg-ink-50` still means "quietest surface", `border-ink-200`
  is still a hairline. **Do not "fix" ink classes when restyling.**
- `brand` is flipped the same way — `brand-600` is the luminous signal
  cyan; `brand-50` a deep cyan wash for selected/active states;
  `text-brand-700/800` for links/accents on dark.
- `bg-panel` (#10151f) is the card fill — replaces every `bg-white`.
- `bg-void` (#07090e) is the page background (already on `<body>`).
- Fonts: `font-sans` = IBM Plex Sans, `font-mono` = IBM Plex Mono,
  `font-display` = Sora (page titles, big readouts, the wordmark).

## Mechanical translation table (light → dark)

| Old (paper theme)                      | New (console theme)            |
|----------------------------------------|--------------------------------|
| `bg-white`                             | `bg-panel`                     |
| `text-white` paired with `bg-brand-600`| `text-ink-50` (dark text on bright cyan) |
| `bg-{c}-50` / `bg-{c}-100` pastel tint | `bg-{c}-500/10` (use `/15` if it needs more presence) |
| `text-{c}-700`/`-800`/`-900` tint text | `text-{c}-300` (or `-400` for secondary) |
| `border-{c}-200` / `-300`              | `border-{c}-500/30`            |
| `ring-{c}-200`                         | `ring-{c}-500/30`              |
| `hover:bg-{c}-50`                      | `hover:bg-{c}-500/10`          |
| `text-red-700` (errors)                | `text-red-400`                 |
| `bg-ink-900` code blocks (`text-ink-100`) | `bg-ink-50 border border-white/5` + `text-ink-800` |

`{c}` = emerald/red/amber/rose/violet/orange/sky/… — every non-ink,
non-brand Tailwind colour. ink/brand utilities are already correct via
the flipped scales. `divide-ink-100`, `shadow-*`, spacing: leave as-is.

## Component classes available (style.css)

- `.panel` / `.panel-pad` — raised card with top sheen.
- `.lamp` + `.lamp-live|ok|fail|idle` — glowing status dots. Use for any
  "is it alive" signal instead of ad-hoc pulsing divs.
- `.pill` + `.pill-blocker|major|minor|nit|status-*` — already dark.
- `.tint-*` — already dark.
- `.btn` / `.btn-primary` / `.btn-ghost` / `.btn-danger` — already dark.
- `.input` / `.select` / `.textarea` / `.label` — already dark.
- `.readout` — mono tabular-nums telemetry text (token counts, durations).
- `.section` / `.section-head` / `.field-grid` — titled form cards.
- `.persona-grid` / `.persona-card` — picker cards.
- `.prose-qa` — markdown body, already dark.
- `.cta-bar` / `.cta-start` — sticky glassy action bar.

## Rules

1. Never introduce a new accent colour. Actions are cyan, period.
2. Severity glyphs stay: ◆ blocker, ▲ major, · minor/nit; 🐞 bug, 👍
   praise, 🔎 observation kind icons stay.
3. Run IDs, action ids, URLs, token counts: `font-mono`.
4. Page `<h1>` is `font-display` (automatic via base styles).
5. Keep every existing `data-testid`. Adding new ones is fine.
6. Empty states get one quiet sentence + one cyan link to the action
   that fills them — no giant illustrations.
7. Headers/labels in tables: `text-xs uppercase tracking-wide text-ink-500`.
8. Hover affordance on rows: `hover:bg-brand-50/40` (deep cyan wash).
